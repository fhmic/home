Forward-Looking Scenario Weights (IFRS 9 §5.5.17)
IFRS 9 requires ECL to be a probability-weighted estimate considering multiple future economic outcomes. Define three scenarios — Base, Upside, and Downside — with their probability weights (must sum to 100%) and macro adjustments relative to your base case. The final weighted ECL = (Base ECL × base%) + (Upside ECL × upside%) + (Downside ECL × downside%).

Historical Loss Rate Calibration
Derive provision matrix rates from your own write-off history (IFRS 9 §5.5.20)
This tab analyses your actual write-off history to compute implied historical loss rates per ageing bucket. These rates represent what percentage of invoices in each bucket have historically become uncollectable. You can then apply these rates directly to your Provision Matrix for a fully entity-specific, audit-defensible ECL model. The more write-off data you accumulate over time, the more reliable these rates become.

Assumption Log & Model Documentation
IFRS 7 §35G disclosure — sensitivity analysis and model documentation
This generates a formal model documentation report capturing all methodology choices, parameter values, scenario weights, and a sensitivity analysis showing how ECL changes under ±1%, ±2%, and ±5% shifts in provision matrix rates. This document supports IFRS 7 paragraph 35G disclosure requirements and provides auditors with the evidence needed to review the ECL model. You must run Compute ECL first before generating the log.
📋
Run Compute ECL on the Portfolio tab, then click Generate Log.
